# Sena Repo - GalGame私有库管理器
## 📱 简介

**Sena Repo** 是一款面向多平台的Galgame/视觉小说私有库管理工具，适合用于管理属于用户自己的远程服务器库（例如NAS里）的游戏，使得使用者能方便地从自己的游戏库中快速下载并导入游戏。

它的目标是把“游戏库管理、快捷下载、数据同步”整合到一个统一的界面中。

通过docker部署服务端刮削整理元数据，然后通过客户端查看并下载。

客户端支持Windows，Android，linux多平台



## 导入以及清洗逻辑

### 目录结构

```
根目录/
  ├── 会社A/                  ← 第一级：会社文件夹
  │   ├── 游戏1/              ← 第二级：游戏文件夹
  │   │   └── [PC]游戏1.rar
  │   └── 游戏2/              ← 同时包含多个平台版本
  │       ├── [PC]游戏2.zip
  │       ├── [KRKR]游戏2.zip
  └── 会社B/
      └── 游戏3/
          └── [ONS]游戏3.7z
```

### 处理流程

**扫描 → 清洗 → 导入 → 刮削**

### 1. 扫描

- **第一级** → 识别为**会社**，自动作为标签附加到该目录下所有游戏
- **第二级** → 识别为**游戏项目**
- **第三级** → 识别为游戏对应的**压缩包文件**，同一游戏文件夹内可包含多个不同平台的版本

### 2. 清洗

从压缩包文件名中通过正则提取平台类型和游戏信息：

| 文件名 | 游戏名 | 平台 | 其他 |
|-------|-------|------|------|
| `[PC]游戏1.rar` | 游戏1 | PC | — |
| `[KRKR]游戏2.zip` | 游戏2 | KRKR | — |
| `[ONS]游戏3.7z` | 游戏3 | ONS | — |
| `[Ty]游戏4.zip` | 游戏4 | Tyranor | — |
| `直装_游戏5.apk` | 游戏5 | 直装 | — |

支持的平台标识：`PC`、`KRKR`、`Ty`（Tyranor）、`ONS`、`直装`

> 末尾为 `.apk` 或文件名含"安卓""直装"字样的自动归类为直装版。

### 3. 导入

- 同一游戏下的不同平台压缩包各生成独立条目，归入同一游戏名下（如 "游戏2" 下同时存在 PC 版和 KRKR 版）
- 非压缩包文件（图片、文档等）自动过滤
- 会社名自动作为标签添加

### 4. 刮削

用游戏名 + 会社标签去刮削源匹配元数据，多信息组合提高准确率。



## 主要功能
- **根目录管理**
  - 添加/删除多个游戏根目录，将根目录下次级文件全部作为游戏项目导入。
  - 一键刷新（扫描根目录并自动刮削更新的游戏）

- **游戏库浏览**
  - 网格、列表展示游戏列表，默认按导入时间倒序排列
  - **搜索**：按游戏名称、路径、标签快速检索
  - **标签系统**：创建/编辑/删除标签，给游戏添加/移除标签


- **封面管理**
  - 手动选择封面文件更新
  - 增加截屏并作为封面按钮
  - 刮削源：

    VNDB Kana v2（免认证）
    
    Bangumi（免认证，填 Token 提速率）
    
    Steam（免认证）
    
    muyueGalgame（免认证）
    
    DLsite（免认证，最好有日本代理）
    
    SteamGridDB（需要 Key）
    
    IGDB（需要 Client ID/Secret）
    
    VNDB（需要 Token）
    
  - **批量刮削封面、厂商名**：对缺失封面的游戏进行自动抓取，刮削得到的厂商名作为标签添加在游戏上
  - **刮削范围配置**：可选择“根目录参与批量刮削”，新增根目录默认勾选
  - **自定义正则**：可使用自定义正则清洗游戏文件名

- **Steam 补丁自动注入**
  - 选择本地 Steam 的 `steamapps/common` 目录，自动识别库内游戏对应的补丁文件
  - 下载补丁后自动解压并注入到游戏目录
  - 支持批量检测和更新

- **背景图片**
  - 支持自定义项目背景图（jpg/png）
  - 可清除背景恢复默认
 
- 封面截图，直接截图作为游戏封面
- 快速定位游戏文件
- 打开进入游戏所在文件夹
- 删除游戏条目（删除后不删除本地文件，只添加进ignore名单，可在配置窗口内恢复）
 

## 部署与安装

### 环境要求

- **服务端**：Docker 20.10+（推荐），或 Python 3.10+ 直接部署
- **客户端**：Windows 10+ / Android 8.0+ / Linux（x86_64）

### 服务端部署

**方式一：Docker 命令行**

```bash
docker run -d \
  --name sena-repo \
  -p 11451:11451 \
  -v /path/to/your/games:/games \
  -v /path/to/data:/data \
  -e SENA_BANGUMI_TOKEN="your_token" \
  sena-repo:latest
```

> 刮削 API Key 通过环境变量传入（可选，不填也能用免认证源）：

| 环境变量 | 对应刮削源 | 获取地址 |
|---|---|---|
| `SENA_BANGUMI_TOKEN` | Bangumi | https://bgm.tv/dev/app |
| `SENA_VNDB_TOKEN` | VNDB | https://vndb.org/u/tokens |
| `SENA_STEAMGRIDDB_KEY` | SteamGridDB | https://www.steamgriddb.com/profile/preferences/api |
| `SENA_IGDB_CLIENT_ID` | IGDB | https://dev.twitch.tv/console/apps |
| `SENA_IGDB_CLIENT_SECRET` | IGDB | 同上 |

**方式二：Docker Compose**

```yaml
version: "3"
services:
  sena-repo:
    image: sena-repo:latest
    container_name: sena-repo
    ports:
      - "11451:11451"
    volumes:
      - /path/to/your/games:/games
      - /path/to/data:/data
    environment:
      - SENA_BANGUMI_TOKEN=your_token
      - SENA_VNDB_TOKEN=your_token
      # 更多环境变量见上方表格
    restart: unless-stopped
```

**方式三：直接部署（不依赖 Docker）**

```bash
# 1. 安装 Python 3.10+
# 2. 克隆项目并安装依赖
git clone <repo-url> && cd sena-repo/server
pip install -r requirements.txt

# 3. 配置（可选，也可用命令行参数）
cp config.example.yaml config.yaml
# 编辑 config.yaml，设置 games_path 和 data_path

# 4. 启动服务端
python main.py --host 0.0.0.0 --port 11451 --games-path /path/to/your/games --data-path /path/to/data
```

> 直接部署时需要自行确保 Python 环境正确，推荐使用 `venv` 或 `conda` 隔离依赖。

启动后可通过 `http://<服务器IP>:11451` 访问管理界面。服务端会自动对 `/games` 目录下的游戏文件进行刮削整理。

### 客户端安装

从 [Releases]() 页面下载对应平台的最新版本：

- **Windows**：下载 `.exe` 安装包，双击安装
- **Android**：下载 `.apk` 文件，允许未知来源后安装
- **Linux**：下载 `.AppImage` 或 `.deb` 包安装

首次启动客户端需填入服务端地址（如 `http://192.168.1.100:11451`），连接成功后即可浏览和下载游戏。

## 使用方法

### 连接服务端

客户端启动后填入服务端地址（如 `http://192.168.1.100:11451`），连接成功后进入游戏库主页。

### 管理根目录

1. 进入设置 → 根目录管理，添加服务端上的游戏存放目录
2. 点击刷新，服务端扫描目录并自动刮削新游戏
3. 可添加多个根目录，支持勾选是否参与批量刮削

### 浏览游戏库

- 支持**网格 / 列表**两种视图，默认按导入时间倒序
- **搜索**：按游戏名称、路径、标签快速检索
- 点击游戏进入详情页，可查看封面、标签、元数据等信息

### 下载游戏

在游戏详情页选择对应平台的压缩包下载，客户端本地目录结构与服务端保持一致（会社/游戏名/压缩包）。

### 封面刮削

- **手动刮削**：在游戏详情页选择刮削源，抓取封面
- **批量刮削**：一键补全所有缺失封面，刮削得到的厂商名自动添加为标签
- **自定义正则**：可配置清洗规则，从文件名中提取更准确的搜索关键词

### 标签管理

创建 / 编辑 / 删除标签，为游戏打标签分类，方便检索筛选。

### Steam 补丁注入

在设置中选择 Steam 游戏安装目录下的 `steamapps/common` 目录，客户端会自动匹配库内游戏对应的补丁，下载后自动解压注入。

### 其他操作

- **修改封面**：手动选择图片替换，或直接截图作为封面
- **背景图片**：自定义游戏项目背景图，可清除恢复默认
- **定位文件**：快速打开游戏所在文件夹
- **删除条目**：删除后不删除本地文件，仅加入 ignore 名单，可在设置中恢复
