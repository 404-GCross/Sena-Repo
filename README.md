# Sena Repo - GalGame私有库管理器

## 📱 简介

**Sena Repo** 是一款面向多平台的视觉小说私有库管理工具，适合管理部署在远程服务器（如 NAS）上的游戏，让使用者能方便地浏览、搜索和下载并安装自己的游戏收藏。

它的目标是把"游戏库管理、元数据刮削、快捷下载与安装"整合到一个统一的界面中。

通过 Docker 部署服务端，服务端负责扫描目录、清洗文件名、刮削元数据；客户端（Windows / Android / Linux）通过 HTTP 连接服务端，提供一体化的游戏库浏览和元数据编辑体验。

## 导入及清洗逻辑

### 目录结构

```
根目录/
  ├── 会社A/                    ← 第一级：会社文件夹
  │   ├── 游戏1/                ← 第二级：游戏文件夹
  │   │   ├── [PC]游戏1.rar
  │   │   └── [Ty]游戏1.zip
  │   ├── 游戏2/                ← 同时包含多个平台版本
  │   │   ├── [PC]游戏2.zip
  │   │   └── [KRKR]游戏2.zip
  │   └── 游戏3/
  │       └── 直装_游戏3.apk
  └── 会社B/
      └── 游戏4/
          └── 游戏4安卓直装版.apk
```

### 处理流程

**扫描 → 清洗 → 导入 → 刮削**

### 1. 扫描

- **第一级** → 识别为**会社**，自动作为标签附加到该目录下所有游戏，同时填入**开发商**字段
- **第二级** → 识别为**游戏项目**
- **第三级** → 识别为游戏对应的**压缩包文件**，同一游戏文件夹内可包含多个不同平台的版本

### 2. 清洗

从压缩包文件名中通过正则提取平台类型和游戏信息：

| 文件名 | 游戏名 | 平台 | 其他 |
|-------|-------|------|------|
| `[PC]游戏1.rar` | 游戏1 | PC | — |
| `[KRKR]游戏2.zip` | 游戏2 | KRKR | — |
| `[ONS]游戏3.7z` | 游戏3 | ONS | — |
| `[Ty]游戏4.zip` | 游戏4 | Tyranor | — |
| `直装_游戏5.apk` | 游戏5 | 安卓直装 | — |
| `Kanon安卓直装版1.0.3.apk` | Kanon | 直装 | 模糊匹配 |
| `<任意名称>.apk` | 任意名称 | 安卓直装 | .apk兜底 |

支持的平台标识：`PC`、`KRKR`、`Ty`（Tyranor）、`ONS`、`直装`

> 末尾为 `.apk` 或文件名含"安卓""直装"字样的自动归类为直装版。

### 3. 导入

- 会社文件夹名自动填入游戏的**开发商**字段（已有游戏只在为空时补填，不覆盖手动修改）
- 同一游戏下的不同平台压缩包各生成独立版本条目，归入同一游戏名下
- 非压缩包文件（图片、文档等）自动过滤
- 会社名自动作为标签添加

### 4. 刮削

用游戏名 + 会社标签去刮削源匹配元数据。所有刮削源均免认证：

| 刮削源 | 说明 |
|--------|------|
| VNDB Kana v2 | 免认证，中文标题 |
| VNDB Titles | 免认证，英文/日文标题 |
| Bangumi | 免认证 |
| Steam | 免认证 |
| DLsite | 免认证（建议配日本代理） |

> IGDB 可选启用，需要配置 Client ID / Secret（从 [Twitch](https://dev.twitch.tv/console/apps) 获取）。

## 主要功能

- **游戏库浏览**
  - 网格 / 列表双视图，支持封面大小自定义
  - 搜索：按游戏名称、路径、标签快速检索
  - 按会社名、导入时间等排序
  - 游戏卡片显示会社名，悬停放大效果（PC）

- **元数据编辑**（一体化）
  - **下载元数据**：选择刮削源 → 搜索 → 对比 → 勾选应用
  - 对比界面可逐字段勾选（名称、开发商、日期、简介、封面）
  - 支持重新选择来源
  - 手动编辑所有字段并保存

- **用户系统**
  - 登录 / 注册，注册需管理员审批
  - 首个注册用户自动激活为管理员
  - 管理员可在设置中审批 / 拒绝注册申请
  - 通知系统：审批请求 + 主界面角标

- **根目录管理**
  - 添加 / 删除多个游戏根目录
  - 一键扫描刷新，支持批量刮削
  - 目录结构选择（会社/游戏 / 仅游戏 / 扁平）

- **标签系统**
  - 会社名自动标签，也可手动创建 / 删除标签
  - 按标签筛选游戏

- **Steam 补丁**（PC 专属）
  - 选择本地 Steam 库目录，自动匹配补丁
  - 下载解压注入流程

- **美化设置**
  - 自定义背景图片（本地文件或 URL）
  - 自定义主题色
  - 客户端全局视觉美化

- **其他**
  - 游戏详情页卡片化布局，封面阴影，平台彩色标签
  - 页面切换/导航过渡动画
  - 快速定位游戏文件夹
  - 删除游戏（软删除，仅加入 ignore 名单）

## 部署与安装

### 环境要求

- **服务端**：Docker 20.10+（推荐），或 Python 3.10+ 直接部署
- **客户端**：Windows 10+ / Android 8.0+ / Linux（x86_64）

### 服务端部署

**方式一：Docker 命令行**

```bash
docker run -d \
  --name sena-repo \
  -p 11451:11451 \
  -v /path/to/your/games:/games \
  -v /path/to/data:/data \
  -e SENA_BANGUMI_TOKEN="your_token" \
  sena-repo:latest
```

> 刮削 API Key 通过环境变量传入（全部可选，不填也能正常使用）：

| 环境变量 | 对应刮削源 | 获取地址 |
|---|---|---|
| `SENA_BANGUMI_TOKEN` | Bangumi（提速率） | https://bgm.tv/dev/app |
| `SENA_IGDB_CLIENT_ID` | IGDB（可选启用） | https://dev.twitch.tv/console/apps |
| `SENA_IGDB_CLIENT_SECRET` | IGDB（可选启用） | 同上 |

**方式二：Docker Compose**

```yaml
version: "3"
services:
  sena-repo:
    image: sena-repo:latest
    container_name: sena-repo
    ports:
      - "11451:11451"
    volumes:
      - /path/to/your/games:/games
      - /path/to/data:/data
    environment:
      - SENA_BANGUMI_TOKEN=your_token
      - SENA_VNDB_TOKEN=your_token
      # 更多环境变量见上方表格
    restart: unless-stopped
```

**方式三：直接部署（不依赖 Docker）**

```bash
# 1. 安装 Python 3.10+
# 2. 克隆项目并安装依赖
git clone <repo-url> && cd sena-repo/server
pip install -r requirements.txt

# 3. 配置（可选，也可用命令行参数）
cp config.example.yaml config.yaml
# 编辑 config.yaml，设置 games_path 和 data_path

# 4. 启动服务端
python main.py --host 0.0.0.0 --port 11451 --games-path /path/to/your/games --data-path /path/to/data
```

> 直接部署时需要自行确保 Python 环境正确，推荐使用 `venv` 或 `conda` 隔离依赖。

启动后可通过 `http://<服务器IP>:11451` 访问管理界面。服务端会自动对游戏目录下的游戏文件进行扫描和元数据整理。

### 客户端安装

从 [Releases]() 页面下载对应平台的最新版本：

- **Windows**：下载 `.exe` / `.zip` 安装包
- **Android**：下载 `.apk` 文件，允许未知来源后安装
- **Linux**：下载 `.AppImage`、`.deb` 或 `.rpm` 包

首次启动客户端需填入服务端地址（如 `http://192.168.1.100:11451`），注册或登录后即可使用。

## 使用方法

### 连接与登录

客户端启动后填入服务端地址（如 `http://192.168.1.100:11451`）。首次使用需注册账户，首个注册用户自动激活为管理员，后续注册需管理员在设置中审批。

### 管理根目录

1. 进入设置 → 扫描设置，添加服务端上的游戏存放目录
2. 点击"开始扫描"，服务端扫描目录结构并导入游戏
3. 可添加多个根目录，支持勾选是否参与批量刮削

### 浏览游戏库

- 左侧切换游戏库 / Steam 补丁 / 我的
- 游戏库支持**网格 / 列表**两种视图，默认按导入时间倒序
- 搜索栏按名称实时筛选，显示游戏计数
- 点击游戏进入详情页，查看封面、标签、元数据、版本列表

### 编辑元数据

- 详情页点击编辑进入 一体化编辑器
- 点击"下载元数据"：选择刮削源 → 搜索 → 逐字段对比勾选 → 应用
- 可手动修改所有字段，保存后立即生效

### 下载游戏

在游戏详情页或编辑页查看版本列表，选择对应平台的压缩包下载。客户端下载目录与服务端保持一致的会社/游戏名结构。

### 封面刮削

- **手动刮削**：在详情页或编辑页选择刮削源搜索并下载封面
- **批量刮削**：设置 → 扫描设置 → 批量刮削，一键补全缺失封面
- **自定义正则**：服务端可配置清洗规则，从文件名中提取更准确的搜索关键词

### 标签管理

会社名自动添加为标签，也可手动创建 / 编辑 / 删除标签，方便分类筛选。

### Steam 补丁注入

在设置中选择 Steam 游戏安装目录下的 `steamapps` 目录，客户端自动匹配库内游戏对应的补丁。
