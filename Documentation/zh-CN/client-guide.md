# Sena-Repo 客户端使用说明书

Sena-Repo 客户端用于连接 Sena-Repo 服务端，浏览游戏库、编辑元数据、下载并解压游戏、导入 Steam 非 Steam 游戏，以及注入 Steam 补丁。当前支持 Windows、Android 和 Linux。

## 安装

从 GitHub Releases 下载对应平台的客户端：

| 平台 | 文件 | 说明 |
| --- | --- | --- |
| Windows | `.exe` / `.zip` | 推荐安装版；便携版解压后运行 |
| Android | `.apk` | 需要授予文件访问权限 |
| Linux | `.AppImage` | 适合桌面 Linux / 掌机系统 |

Windows 和 Linux 客户端内置 7zip-zstd 相关解压组件。Android 使用应用内下载与解压流程，Steam 相关功能不在 Android 端显示。

## 首次启动

客户端首次启动时会要求填写本机设置：

- 游戏下载目录。
- Steam `steamapps` 目录。
- Steam 用户 ID。

这些设置只保存在当前设备上，不会写入服务端。之后可以在登录/服务器选择界面右上角齿轮，或“设置 -> 下载设置”中修改。

Steam 用户 ID 是 Steam 好友代码，也可以从 Steam 安装目录的 `userdata` 文件夹中找到纯数字目录名。

## 连接服务器

### 添加服务器

在服务器选择界面点击“添加服务器”，进入二级界面填写：

- 配置名称。
- 服务器地址。
- 端口，默认 `11451`。
- 是否使用 HTTPS。
- 用户名和密码。

保存后会作为一个服务器配置显示在列表中。服务器卡片右侧的三个点菜单可编辑或删除配置。编辑服务器地址、端口、协议、用户名或密码后，相当于重新登录该配置。

### 自动登录

客户端会优先尝试自动登录上次使用的服务器配置。启动时会请求 `/api/auth/profile/me` 检查保存的 token 是否仍然有效：

- token 有效：直接进入游戏库。
- token 过期或服务器重置：弹出重新登录或初始化流程。
- 连接异常：显示连接失败弹窗，不会静默失败。

### 退出登录

“我的”界面点击退出登录后会回到服务器选择界面。退出登录只清除当前激活会话，不会删除已保存的服务器配置。

## 首次初始化服务端

连接到一个未初始化的服务端时，客户端会进入服务端初始化向导。

### 1. 创建管理员

填写首个管理员用户名和密码。管理员拥有扫描、目录管理、用户审批、刮削配置等权限。

### 2. 目录与扫描

配置服务端侧目录：

- 游戏库目录。
- Steam 补丁库目录。
- OpenList 服务器。
- 自动扫描开关。
- 扫描间隔。
- 目录结构：会社 / 游戏、仅游戏、扁平。

添加目录时可选择：

- 本地文件源：路径相对于服务端或容器。
- OpenList 源：先添加 OpenList 服务器，再选择该服务器并填写 OpenList 内部目录。

### 3. 刮削源

选择默认启用的刮削源：

- VNDB Kana v2。
- Bangumi。
- Steam。
- YMGal。

还可以配置“批量自动刮削字段来源”。例如名称用 Steam、封面用 VNDB、简介用 Bangumi、游戏时长用 VNDB。该规则只影响批量刮削和自动刮削，单个游戏手动刮削仍由用户在编辑页自行选择。

完成初始化后，服务端会后台扫描目录，客户端进入主界面。

## 游戏库浏览

游戏库支持：

- 网格视图和列表视图。
- 搜索游戏名。
- 按平台筛选。
- 按标签筛选。
- 排序。
- 批量选择。

长按或进入多选模式后，可以对多个游戏执行批量操作，例如批量刮削或删除。批量删除是软删除，不会删除本地文件或 OpenList 文件。

## 游戏详情与编辑

点击游戏进入详情页，可查看：

- 封面、背景图。
- 游戏名、会社、开发商。
- 简介。
- 发售日、游戏时长。
- VNDB / Steam / Bangumi ID。
- 标签。
- 版本列表。

### 编辑元数据

详情页进入编辑界面后，可以手动修改：

- 名称。
- 会社 / 开发商。
- 简介。
- 发售日。
- 游戏时长。
- VNDB ID。
- Steam ID。
- Bangumi ID。
- 标签。

### 图片编辑

封面和背景支持：

- 选择本地图片。
- 填写图片 URL 下载到本地临时缓存。
- 保存时上传到服务端。

图片 URL 方式和刮削图片类似：先在客户端预存，确认保存后才写入服务端。

### 手动刮削

编辑界面点击下载元数据，可以搜索多个来源并对比字段。支持直接输入：

- 游戏名。
- VNDB ID。
- Steam ID。
- Bangumi ID。

搜索结果可逐字段勾选应用，应用后仍需点击保存才会写入服务端。

### 版本管理

版本列表右侧菜单支持：

- 移动到其他游戏。
- 修改版本平台，例如 PC、KRKR、Tyranor、ONS、Android 直装。
- 预填解压密码。

预填解压密码后，下载解压时如果遇到需要密码，会自动使用该版本保存的密码重试。

## 下载与解压

### 下载流程

在详情页点击下载版本：

1. 客户端请求服务端下载接口。
2. 本地文件源由服务端返回文件。
3. OpenList 文件源会经过 302 跳转到 OpenList，再跳到网盘直链。
4. 客户端下载到本地下载目录。
5. 下载完成后自动解压。
6. 显示完成弹窗和后续操作按钮。

OpenList 下载流量不经过 Sena-Repo 服务端转发，客户端必须能访问 OpenList 地址和最终网盘直链。

### 下载控制

下载弹窗和下载管理页支持：

- 查看进度、速度和状态。
- 暂停。
- 继续。
- 取消。
- 后台运行。
- 打开目标文件夹。

超过最大同时下载数的任务会进入等待队列，不应该直接失败。

### 下载设置

进入“设置 -> 下载设置”可配置：

- 最大同时下载数，范围 1 到 10。
- 游戏下载目录。
- 桌面快捷方式保存位置。
- 下载限速，单位 KB/s，`0` 表示不限速。

下载限速是客户端总下载速度限制，不是单个任务速度限制。

### Android 权限

Android 下载和解压需要文件访问权限。若未授权，客户端会弹窗提示并跳转系统设置。授权后回到应用会继续之前的下载操作。

## Steam 集成

Windows 和 Linux 端下载解压完成后，可将游戏添加到 Steam 作为非 Steam 游戏。

需要配置：

- Steam `steamapps` 目录。
- Steam 用户 ID。

客户端会写入 Steam `shortcuts.vdf`，并尽量导入封面或背景图。添加后通常需要重启 Steam 才能看到完整变化。

解压完成弹窗中的按钮会根据平台和状态显示，例如：

- 打开目标文件夹。
- 添加到 Steam。
- 创建快捷方式。

Android 端不提供 Steam 集成功能。

## Steam 补丁注入

Steam 补丁页仅在 Windows / Linux 显示。

### 使用流程

1. 服务端扫描补丁库。
2. 客户端选择本机 Steam `steamapps` 目录。
3. 客户端扫描已安装 Steam 游戏。
4. 按 Steam AppID 匹配服务端补丁。
5. 点击注入。
6. 客户端下载补丁、解压并覆盖到对应游戏目录。

注入后可打开对应游戏目录检查结果。

### 服务端 Tab

服务端 Tab 用于管理补丁索引：

- 加载补丁列表。
- 扫描补丁。
- 批量刮削 AppID。
- 单个补丁重新刮削。
- 编辑 AppID、补丁类型、目标目录等。
- 编辑补丁类型关键字。

### 客户端 Tab

客户端 Tab 用于本机匹配和注入：

- 选择 Steam 库目录。
- 扫描本地 Steam 游戏。
- 查看可注入补丁。
- 开始注入、暂停、取消。

## 设置

### 下载设置

用于配置本机下载相关选项：

- 下载目录。
- 最大同时下载数。
- 下载限速。
- 快捷方式保存位置。

### 扫描设置

管理员可进入“设置 -> 扫描设置”管理服务端侧扫描：

- OpenList 服务器。
- 游戏库目录。
- Steam 补丁库目录。
- 自动扫描。
- 目录结构。
- 立即扫描。
- 清空游戏库并重新扫描。
- 批量刮削。
- 刮削源。
- 批量自动刮削字段来源。

“清空并重扫”会清除数据库里的游戏条目、版本、分类关联和刮削数据，然后重新扫描所有游戏库目录。它不会删除实际文件，也不会删除目录设置或用户账号。

### 显示设置

可调整：

- 封面显示大小。
- 托盘相关行为。
- 外观偏好。

### 客户端本地设置

登录/服务器选择界面右上角齿轮可修改首次启动时填写的本机目录设置，包括下载目录、Steam `steamapps` 目录和 Steam 用户 ID。

## 用户与通知

### 注册

登录界面可注册新用户。新用户默认需要管理员审批，通过后才能正常登录。

### 管理员审批

管理员可在：

- 通知页审批注册请求。
- 设置 -> 用户管理中审批或拒绝用户。

审批完成后，对应审批通知会标记为已读，避免通知红点残留。

### 用户管理

管理员可以：

- 查看用户。
- 审批或拒绝注册。
- 授予或取消管理员权限。
- 删除用户。

## Linux 触屏

Linux 端针对 Wayland、X11、Gamescope、Bazzite 等环境做了触屏兼容处理。启动 AppImage 时可用终端查看日志：

```bash
./Sena-Repo_Linux_v0.1.4-x86_64.AppImage 2>&1 | tee sena-touch.log
```

正常情况下，触摸屏输入会出现类似日志：

```text
Sena Linux GTK display: wayland-0
Sena Linux touch event: begin x=...
Sena Linux touch event: update x=...
```

如果完全没有 touch event 日志，说明系统或桌面环境没有把触摸输入传给应用。若有日志但界面不响应，优先反馈日志中的 backend、widget 和触摸坐标。

## 排障

### 点击服务器无反应

检查：

1. 服务器地址和端口是否正确。
2. 是否填写了 `http://` / `https://` 对应协议。
3. 服务端 `/api/health` 是否可访问。
4. token 是否过期，必要时从服务器配置菜单重新登录。
5. 如果服务端已重置，进入初始化流程重新设置。

### 下载报 401

通常是保存的登录 token 已失效。回到服务器选择界面，点击对应配置重新登录。客户端启动时会主动检查 token，但服务端重建、数据库替换或旧版本升级后仍可能需要重新登录。

### 下载卡住或速度慢

检查：

1. 下载设置是否启用了限速。
2. 当前是否有多个任务占用总速度。
3. OpenList 地址是否客户端可访问。
4. 客户端日志中的下载跳转是否到达最终网盘 CDN。
5. 浏览器直接下载同一个 OpenList 文件是否正常。

### OpenList 扫描不到游戏

确认：

- OpenList 服务器已先添加。
- 游戏库目录选择的是 OpenList 源。
- 填写的是 OpenList 内部路径，例如 `/115/Games/GalGame/Library`。
- 目录结构选择正确。
- 目标路径下还有会社目录、游戏目录和压缩包。

### Android 安装 APK 失败

如果报错类似 `Failed to find configured root that contains ...apk`，通常是安装来源路径不在应用可共享的文件目录下。优先把下载目录设到 Android 公共下载或应用有权限访问的位置，并确认已授予文件访问权限。

### Linux 触控仍不生效

先用终端启动并保存日志。如果日志里能看到 `Sena Linux touch event`，说明应用收到事件；如果没有，优先检查系统输入、Gamescope/Wayland 设置或 AppImage 运行环境。
